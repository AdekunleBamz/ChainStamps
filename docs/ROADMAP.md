# ChainStamps Roadmap

This roadmap outlines potential improvements and features. Items are not guaranteed and may evolve based on community feedback.

## Short Term (0-3 months)

- [ ] Add pagination helpers for user hashes/stamps/tags
- [ ] Provide contract versioning in read-only functions
- [ ] Publish example integrations for web and CLI
- [ ] Expand contract tests with negative cases
- [ ] Improve documentation coverage (API reference, FAQs)

## Mid Term (3-6 months)

- [ ] Batch operations for higher throughput
- [ ] Optional metadata fields for stamps and tags
- [ ] Indexing helpers for off-chain services
- [ ] Event-style logs for indexing tools
- [ ] SDK helper library for common patterns

## Long Term (6-12 months)

- [ ] Multi-contract registry for project management
- [ ] Preset templates for common use cases
- [ ] Analytics dashboard for on-chain usage
- [ ] Governance for fee updates and contract upgrades
- [ ] Cross-chain verification adapters

## Community Contributions

If you'd like to contribute, please open an issue or PR. Feedback is welcome.
