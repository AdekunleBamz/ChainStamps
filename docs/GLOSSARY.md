# ChainStamps Glossary

## A

**Anchor Block**
A Bitcoin block that anchors a Stacks block, providing Bitcoin-level security and finality.

**API (Stacks API)**
The public endpoint used to query blockchain data and broadcast transactions.

## B

**Block Height**
The sequential number of a block in the blockchain.

**Broadcast Transaction**
The process of submitting a signed transaction to the Stacks network.

## C

**ChainStamp**
A timestamped record stored on the Stacks blockchain via the stamp-registry contract.

**Clarity**
Stacks smart contract language that is decidable and interpreted.

**Contract Owner**
The principal that deployed the contract, designated to receive fees.

## D

**Decentralization**
A distributed system with no single controlling entity.

## F

**Fee**
A required STX payment for storing data on-chain.

## H

**Hash**
A cryptographic digest (SHA-256) representing document content.

**Hash Registry**
The contract that stores document hashes with metadata.

## I

**Immutability**
Data cannot be modified or deleted after it is written to the blockchain.

## M

**Mainnet**
The production Stacks network using real STX tokens.

## P

**Principal**
A Stacks identity (user or contract address) used in transactions.

## S

**Stamp Registry**
The contract that stores short messages and timestamps on-chain.

**Stacks**
A Bitcoin layer-2 blockchain enabling smart contracts and apps.

**STX**
The native token used for transaction fees and contract fees on Stacks.

## T

**Tag Registry**
The contract that stores key-value tags with ownership and updates.

**Testnet**
A Stacks network for development and testing using test STX.

## U

**UTF-8**
A text encoding format used for contract strings.

## V

**Verification**
The process of checking if a hash or stamp exists on-chain.
